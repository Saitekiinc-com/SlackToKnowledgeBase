# ナレッジグラフ登録機能

## 1. 機能概要
`knowledge/`ディレクトリ内のMarkdownファイルが追加または更新された際に、ファイル内容（メタデータ・本文）を解析し、LLMを用いてキーワード（Entity）を抽出した上で、Neo4j（グラフデータベース）にナレッジ間のつながり（ナレッジグラフ）を構築する機能です。

## 2. 実行トリガー
- **イベント**: GitHub Actionsの `push` イベント
- **対象ブランチ**: `main`
- **対象パス**: `knowledge/**/*.md` (ナレッジファイルの変更時のみ起動)

## 3. 処理フロー
処理は `.github/workflows/register-graph.yml` と `scripts/register_graph.py` によって実行されます。

1. **変更ファイルの特定**:
   - 直前のコミット（`HEAD~1`）との差分を比較し、追加・変更された `.md` ファイルのみをリストアップします。対象がない場合は処理をスキップします。
2. **既存Entityの取得**:
   - 語彙のブレを防ぐため、Neo4jから既に登録されているEntity（キーワード）の一覧を事前に取得します。
3. **Markdownの解析 (パース)**:
   - 対象ファイルの Frontmatter から `title`, `date`, `author` を取得します。
   - 本文から `## 背景、課題`, `## 手段`, `## 結果`, `## ツールやAIのモデル` の各セクション内容を抽出します。
4. **LLMによるエンティティ（キーワード）抽出**:
   - 本文テキストと、ステップ2で取得した「既存Entity一覧」をLLM（OpenAI GPT-4o-mini）に渡し、5〜10個の重要なキーワード（AI手法、ツール、用途、性質など）を抽出させます。既存キーワードと重複する場合は既存の表記に統一されます。
5. **Neo4jへのグラフ登録**:
   - ファイルのパスを元に一意の `knowledge_id` (`knowledge__YYYY-MM-DD-HHMMSS-title`) を生成します。
   - **ノード生成**:
     - `Knowledge`ノード（ナレッジ本体）を作成し、タイトルや本文各セクションのプロパティをセットします。
     - `Author`ノード、`Tool`ノード、`Entity`ノードを作成（または既存とマージ）します。
   - **エッジ（リレーション）生成**:
     - `WRITTEN_BY` (Knowledge → Author)
     - `USES_TOOL` (Knowledge → Tool)
     - `CONTAINS_ENTITY` (Knowledge → Entity)
   - **自動関連付け (RELATED_TO)**:
     - 登録したナレッジと共通の `Entity` を2つ以上持つ他の有効な（`status='active'`）ナレッジを検索し、それらの間に `RELATED_TO` エッジ（重み付き）を自動生成します。

## 4. 出力内容（Neo4j データ構造）
- **Nodes**: `Knowledge`, `Entity`, `Tool`, `Author`
- **Relationships**: `CONTAINS_ENTITY`, `USES_TOOL`, `WRITTEN_BY`, `RELATED_TO`

## 5. エラーハンドリング
- LLMによるエンティティ抽出が失敗した場合でも、`Knowledge` / `Author` / `Tool` ノードの登録処理は継続されます。
- エラーが発生したファイルについては後続のログに記録し、GitHub Actions は失敗（Error）として終了し、必要に応じて再実行が可能なように設計されています。
