# AIナレッジ管理システム 環境設定

本ナレッジベース（投稿・登録・検索・陳腐化チェック機能）を運用する上で必要な環境設定を以下にまとめます。

---

## ① GitHub の Repository Secrets / Variables

GitHub リポジトリの `Settings > Secrets and variables > Actions` にて設定が必要な環境変数（Secrets / Variables）です。

### Secrets (機密情報)

| Secret名 | 説明 | 利用される機能・用途 |
|---|---|---|
| `OPENAI_API_KEY` | OpenAI APIの認証キー | ナレッジグラフ登録機能（エンティティ抽出）、ナレッジ検索機能（回答生成） |
| `NEO4J_URI` | Neo4j データベースの接続URI | ナレッジグラフ登録機能、ナレッジ検索機能、陳腐化チェック機能（データ削除） |
| `NEO4J_USERNAME` | Neo4j のユーザー名 | 同上 |
| `NEO4J_PASSWORD` | Neo4j のパスワード | 同上 |
| `SLACK_WEBHOOK_URL` | Slack Incoming Webhook URL | ナレッジ検索結果の通知、陳腐化チェック時の確認・無効化通知、エラー通知 |
| `AUTHOR_SLACK_MAP` | GitHubアカウント名とSlackユーザーIDのマッピング（JSON形式） | 陳腐化チェック機能での個人宛メンション（例: `{"github_user": "U12345678"}`） |

> [!NOTE]
> `GITHUB_TOKEN` はGitHub Actionsによって自動で一時生成されるトークンを使用するため、通常は個別登録不要ですが、外部（Cloudflare等）からワークフローを起動する場合には Personal Access Token (PAT) を発行する必要があります。

### Variables (環境変数)

| Variable名 | 説明 | 利用される機能・用途 | デフォルト値 |
|---|---|---|---|
| `KNOWLEDGE_CHECK_DAYS` | ナレッジ投稿日から確認対象(`check/`)へ移動するまでの日数 | 陳腐化チェック機能 | 30 |
| `KNOWLEDGE_INVALID_DAYS` | ナレッジ投稿日から無効化(`invalid/`)されるまでの日数 | 陳腐化チェック機能 | 90 |

## ② Webhook (Cloudflare Workers 系)

Slackの操作（スラッシュコマンド等）を受け付け、GitHub Actions (`repository_dispatch`) を起動するための中継役として、Cloudflare Workers を設定します（スクリプト: `/work/cloudflare-worker-v2.js`）。

### Cloudflare Workers 環境変数 (Settings > Variables)

| 変数名 | 説明 |
|---|---|
| `SLACK_BOT_TOKEN` | Slack AppのBot User OAuth Token (`xoxb-...`)。SlackにModal画面を表示したり、処理受付メッセージを送信するのに使用します。 |
| `SLACK_SIGNING_SECRET` | Slackからのリクエストの署名検証用シークレット。不正なリクエストを弾くために使用します。 |
| `GITHUB_TOKEN` | GitHub Personal Access Token (PAT)。`repo` 権限など、`repository_dispatch` イベントを発火できる権限が必要です。 |
| `GITHUB_OWNER` | 対象のGitHubユーザー名 または Organization名 |
| `GITHUB_REPO` | 対象のリポジトリ名 |

## ③ Slack アプリ設定

Cloudflare Workers と連携するための Slack App の設定項目です。Slack API ダッシュボードから設定します。

### 1. Interactivity & Shortcuts
ユーザーのアクション（ボタン押下、Modal送信など）を処理するための設定です。
- **Interactivity**: `On`
- **Request URL**: Cloudflare Worker の URL

### 2. Slash Commands
ユーザーがチャット欄から機能を呼び出すためのコマンド設定です。
- **Command**: `/knowledge`
  - **Request URL**: Cloudflare Worker の URL
  - **Description**: ナレッジ投稿Modalを開く
- **Command**: `/knowledge-search` (または `/search`)
  - **Request URL**: Cloudflare Worker の URL
  - **Description**: ナレッジ検索Modalを開く

### 3. OAuth & Permissions (Bot Token Scopes)
Slack Appに必要な権限（スコープ）です。
- `commands`: スラッシュコマンドを追加するために必要
- `chat:write`: 検索受付時や投稿完了時に、Botからメッセージを送信するために必要

### 4. Event Subscriptions (設定がある場合)
※本構成では主にSlash CommandsとInteractivity (Modal送信) で動作するため、Event APIの購読は必須ではありません。
