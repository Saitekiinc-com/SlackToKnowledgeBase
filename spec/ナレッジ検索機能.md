# ナレッジ検索機能

## 1. 機能概要
ユーザーからの自然言語での質問（クエリ）を受け取り、Neo4jナレッジグラフを検索して関連するナレッジ情報を取得し、LLMを活用して回答を生成してSlackまたはGitHub Issueに返信する機能です。
検索ロジックは共通化されており、呼び出し元インターフェース（SlackまたはGitHub）に応じて出力先が分岐します。

## 2. 実行トリガー

### 2.1 Slackからの呼び出し
- **イベント**: `repository_dispatch`
- **タイプ**: `search-knowledge-slack`
- **実行環境**: GitHub Actions (`.github/workflows/search-knowledge-slack.yml`)
- **ペイロード**: `query` (検索クエリ文字列), `channel_id` (応答先SlackチャンネルID)

### 2.2 GitHub Issueからの呼び出し
- **イベント**: `issue_comment` (Issueへのコメント追加時)
- **条件**: コメントが `/search ` から始まる場合
- **実行環境**: GitHub Actions (`.github/workflows/search-knowledge-github-issue.yml`)
- **ペイロード**: コメント本文（`/search ` 以降の文字列をクエリとして抽出）

## 3. 処理フロー

### 3.1 共通検索ロジック (`scripts/search_graph_common.py`)
1. **クエリからのエンティティ抽出**:
   - ユーザーのクエリをLLM (GPT-4o-mini) に渡し、検索キーワード（エンティティリスト）を抽出します。
2. **Neo4j グラフ検索**:
   - 抽出したエンティティを元にNeo4jで `MATCH (e:Entity)<-[:CONTAINS_ENTITY]-(k:Knowledge)` のCypherクエリを発行します。
   - キーワードと一致するナレッジをヒット数（スコア）順に取得します。
   - ヒットしたナレッジに関連する他のナレッジ（`RELATED_TO`エッジ）も辿って追加の文脈（周辺ナレッジのタイトル）を取得します。
3. **回答の生成 (RAG)**:
   - 取得したナレッジ情報（タイトル、背景、手段、結果、ツール、関連ナレッジ）をコンテキストとしてLLMに渡し、ユーザーの質問に対する最終的な回答文章を生成させます。
   - 参考にしたナレッジのタイトル一覧も末尾にリストアップされます。

### 3.2 出力処理

#### Slack出力 (`scripts/search_graph_slack.py`)
- 共通ロジックで生成した回答テキストを受け取ります。
- Slack Incoming Webhook を使用し、指定された `channel_id` に対してテキストメッセージを投稿します。

#### GitHub Issue出力 (`scripts/search_graph_github.py`)
- 共通ロジックで生成した回答テキストを受け取ります。
- PyGithub を使用し、呼び出し元のIssue (`ISSUE_NUMBER`) に対して、生成された回答をIssueコメントとして投稿します。

## 4. エラーハンドリング
- Slack/GitHub どちらのトリガーにおいても、ワークフロー実行中にエラー（スクリプトの異常終了など）が発生した場合、失敗ステップが実行され、Slackの指定チャンネルに対してエラー発生通知とログのURLが送信されます。
