# ナレッジ投稿機能

## 1. 機能概要
Slackからのリクエストをトリガーとし、入力されたナレッジ情報をMarkdownファイルに変換してGitHubリポジトリ（`knowledge/`ディレクトリ）に保存し、自動的にPull Requestを作成する機能です。

## 2. 実行トリガー
- **イベント**: `repository_dispatch`
- **タイプ**: `slack-knowledge-submitted`
- **呼び出し元**: SlackのワークフロービルダーなどからWebhook（GitHub API）経由で実行

## 3. 入力内容 (Client Payload)
Slackから送られてくるペイロードデータは以下のフィールドを持ちます。
- `title` (必須): ナレッジのタイトル
- `why` (必須): 背景、課題
- `how` (必須): 手段、やり方
- `result` (必須): 結果
- `tools` (必須): 使用ツールやAIモデル
- `author` (必須): 投稿者名

## 4. 処理フロー
処理の本体は `.github/workflows/save-knowledge.yml` に記述されています。

1. **ペイロードの受け取り**:
   - `github.event.client_payload` から各フィールドの値を変数として取得します。
2. **ファイル名と日時の生成**:
   - 実行時の日本時間（JST）を取得し、`knowledge/YYYY-MM-DD-HHMMSS-タイトル.md` の形式でファイル名を生成します。
   - タイトル内のファイル名に使用できない特殊文字は除去されます。
3. **Markdownの構築**:
   - Frontmatter（メタデータ）として `date`, `title`, `author` をYAML形式で設定します。
   - 本文は以下の見出し構成でフォーマットされます。
     - `## 背景、課題`
     - `## 手段`
     - `## 結果`
     - `## ツールやAIのモデル`
4. **ファイル保存**:
   - `knowledge/` ディレクトリが存在しない場合は作成し、生成したMarkdownを出力します。
5. **Git操作とPRの作成**:
   - `knowledge_YYYYMMDDHHMMSS` という一意のブランチ名を作成してチェックアウトします。
   - 作成したファイルをコミットし、リモートリポジトリへPushします。
   - GitHub CLI (`gh pr create`) を用いて、`main` ブランチに対するPull Requestを自動作成します。

## 5. 出力
- **ファイル**: `knowledge/*.md` ファイル
- **GitHub**: レビュー用の Pull Request
