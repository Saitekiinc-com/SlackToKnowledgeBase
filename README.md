# SlackToKnowledgeBase

Slack のショートカットから入力した内容を Markdown ファイルとして自動保存するリポジトリです。

## フォルダ構成

```
knowledge/   ← Slack から登録したナレッジが保存される
.github/workflows/save-knowledge.yml  ← 自動保存ワークフロー
```

## 使い方

1. メッセージ入力欄のショートカットアイコンをクリック
2. **「ナレッジを登録する」** を選択
3. フォームに入力して送信
4. 自動で PR が作成されるので、レビュー後にマージ

## 設計思想

ナレッジグラフ式のRAG採用につき、各セクションでの自由入力形式（リッチテキスト対応）で投稿する設計としています。

## 入力項目

| 項目 | 必須 | 内容 |
|---|---|---|
| タイトル (title) | ○ | ナレッジのタイトル（日本語対応） |
| 背景、課題 (why) | ○ | ナレッジの背景や解決したい課題 |
| 手段 (how) | ○ | 解決のためのプロンプトや各種アクション |
| 結果 (result) | ○ | 実行した結果や得られた知得 |
| ツールやAIのモデル (tools) | ○ | 使用したAIモデルや利用ツール |

※ 投稿者(`author`)と登録日(`date`)はシステム側で自動で付与されます。

## 保存フォーマット

送信すると `knowledge/` フォルダに以下の形式で Markdown ファイルが自動生成されます。

- **ファイル名**: `YYYY-MM-DD-HHMMSS-タイトル.md`（日本語タイトルをそのまま使用）
- **タイムゾーン**: JST（日本標準時）

```markdown
---
date: "2026-03-05"
title: "ナレッジのタイトル"
author: "著者名"
---

# ナレッジのタイトル

## 背景、課題
背景や課題の内容がここに入ります。

## 手段
手段の内容がここに入ります。

## 結果
結果がここに入ります。

## ツールやAIのモデル
使用したツールがここに入ります。
```

## ワークフロー

ナレッジ送信後、GitHub Actions が以下を自動実行します。

1. タイムスタンプ付きブランチを作成（`knowledge_YYYYMMDDHHMMSS`）
2. Markdown ファイルを生成・コミット
3. main ブランチへの PR を自動作成（タイトル: `○○ の追加マージ依頼`）
