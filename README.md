# SlackToKnowledgeBase

Slack のショートカットから入力した内容を Markdown ファイルとして自動保存するリポジトリです。

## フォルダ構成

```
knowledge/   ← Slack から登録したナレッジが保存される
.github/workflows/save-knowledge.yml  ← 自動保存ワークフロー
```

## 使い方

1. メッセージ入力欄のショートカットアイコンをクリック
2. **「ナレッジを登録する」** を選択
3. フォームに入力して送信
4. 自動で PR が作成されるので、レビュー後にマージ

## 入力項目

| 項目 | 必須 | 内容 |
|---|---|---|
| タイトル | ○ | ナレッジのタイトル（日本語対応） |
| カテゴリ | ○ | 技術・運用 など |
| フェーズ | − | 設計・実装・テスト など |
| AIモデル | − | 使用した AI モデル名 |
| タグ | − | 複数タグをカンマ区切りで指定可 |
| 内容 | ○ | 本文（太字・斜体・リンク・コードブロック対応）|

## 保存フォーマット

送信すると `knowledge/` フォルダに以下の形式で Markdown ファイルが自動生成されます。

- **ファイル名**: `YYYY-MM-DD-HHMMSS-タイトル.md`（日本語タイトルをそのまま使用）
- **タイムゾーン**: JST（日本標準時）

```markdown
---
date: "2026-03-05"
title: "ナレッジのタイトル"
category: "技術"
phase: "実装"
ai_model: "Gemini"
tags: "Python, GitHub Actions"
author: "著者名"
---

# ナレッジのタイトル

本文がここに入ります。
```

## ワークフロー

ナレッジ送信後、GitHub Actions が以下を自動実行します。

1. タイムスタンプ付きブランチを作成（`knowledge_YYYYMMDDHHMMSS`）
2. Markdown ファイルを生成・コミット
3. main ブランチへの PR を自動作成（タイトル: `○○ の追加マージ依頼`）
